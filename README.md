# AutoSchoolProject

Уеб приложение за управление на автошкола, разработено с **ASP.NET Core MVC**.  
Системата улеснява работата на **администратор**, **инструктор** и **курсист**, като обединява на едно място регистрация, заявки за записване, графици, практически часове, съобщения и административни справки.

## Основни функционалности

- регистрация и вход на потребители;
- ролево разделение: **Администратор**, **Инструктор**, **Курсист**;
- подаване и обработка на заявки за записване;
- управление на профили и профилни снимки;
- записване, одобрение, отказ, пренасрочване и отмяна на практически часове;
- графици и съобщения между участниците;
- административни панели за управление на курсисти, инструктори, курсове и справки.

## Използвани технологии

- **C#**
- **ASP.NET Core MVC**
- **Entity Framework Core**
- **ASP.NET Core Identity**
- **Microsoft SQL Server / LocalDB**
- **Razor Views**
- **Bootstrap**
- **HTML / CSS / JavaScript**

## Структура на проекта

- `Controllers` – контролери за основната логика;
- `Models` – основни модели и ентитита;
- `ViewModels` – модели за визуализация;
- `Services` – бизнес логика;
- `Data` – контекст, seed данни и миграции;
- `Views` – Razor изгледи;
- `Areas/Admin` – административна зона;
- `wwwroot` – статични файлове, стилове, скриптове и качени изображения.

## Изисквания

Преди стартиране се увери, че имаш инсталирани:

- **.NET SDK**
- **SQL Server** или **LocalDB**
- **Visual Studio 2022** / **Visual Studio Code** (по желание, но е препоръчително)

## Стартиране на проекта

### 1. Клонирай repository-то

```bash
git clone <repo-url>
cd AutoSchoolProject
```

### 2. Провери connection string-а

Отвори:

- `appsettings.json`
- `appsettings.Development.json` (ако се използва)

и провери дали `DefaultConnection` сочи към твоя SQL Server / LocalDB.

Пример:

```json
"ConnectionStrings": {
  "DefaultConnection": "Server=(localdb)\\MSSQLLocalDB;Database=AutoSchoolProject;Trusted_Connection=True;MultipleActiveResultSets=true"
}
```

### 3. Приложи миграциите

В Package Manager Console или терминал изпълни:

```bash
dotnet ef database update
```

### 4. Стартирай проекта

```bash
dotnet run
```

или го стартирай директно през **Visual Studio**.

## Seed данни

Проектът използва seed данни за:
- роли;

## Потребителски роли

### Администратор
- управлява курсове, курсисти, инструктори и заявки;
- обработва записвания;
- следи часове и справки;
- управлява активни/завършили курсисти и активни/неактивни инструктори.

### Инструктор
- създава свободни слотове;
- приема, отказва и пренасрочва практически часове;
- отбелязва проведени часове;
- следи съобщения и списъци с курсисти.

### Курсист
- редактира профил;
- избира инструктор от своята категория;
- записва практически часове;
- следи график, съобщения и напредък.
